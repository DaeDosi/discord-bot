### 6. 치지직 실시간 채팅 명령어 — 이번 세션의 핵심 신규 기능 (`467d01c` ~ `69da220`)
**요구사항**: 치지직 생방송 채팅에서 `!출석체크`(커스터마이즈 가능)를 치면 하루 1회
포인트+애정도를 지급하고, 그 외 최대 5개까지 등록 가능한 커스텀 명령어는 지정한 문구로
자동 응답.

**아키텍처 결정**: 비공식 쿠키 스크래핑 대신 치지직 **공식 오픈 API의 세션(Session)
기능**을 사용 — 이미 스트리머 연동 시 저장해둔 OAuth 토큰을 재사용. 프로토콜(Engine.IO v3
기반, 표준 `python-socketio`로는 안 됨)은 직접 구현하지 않고 이미 이걸 구현해 둔
[`chzzkpy`](https://pypi.org/project/chzzkpy/) 라이브러리를 실제로 설치해 API 시그니처를
검증한 뒤 사용.

**구현물**:
- DB: `chzzk_chat_commands`(guild당 checkin 1개 + reply 최대 5개), `chzzk_checkin_log`
  (1일1회 강제, PK 충돌로 중복 차단), `chzzk_chat_log`(디버그용 최근 50개 채팅 로그),
  `chzzk_subscriptions.chat_last_sync_at`/`chat_last_event_at`(연결 상태 표시용).
- 스트리머 OAuth 스코프에 "채팅 메시지 조회"/"채팅 메시지 쓰기" 추가 — **기존에 연동된
  스트리머는 재연동 필요**.
- `cogs/chzzk_chat.py`(신규): 세션 연결 → 스트리머별 구독 → 채팅 이벤트 수신 → 명령어
  매칭 → 지급/응답. 5분마다 동기화 + 게이트웨이 생존 확인(끊겼으면 강제 재연결).
- 대시보드 "실시간 채팅 명령어" 탭: 명령어 CRUD, 연결 상태 카드, 실시간 채팅 미리보기
  (기본 숨김, 버튼으로 열람 — 실제 수신 메시지/봇 응답을 그대로 보여주는 디버그 도구).
- 입장 인증 탭에 임베드 미리보기 토글 버튼 추가.

**실전에서 잡은 버그들** (라이브 로그 보고 그 자리에서 수정):
- `Client.connect()`의 반환값이 구독 API가 요구하는 세션 키가 아니라 소켓 transport
  ID였음 → `on_connect` 이벤트로 진짜 세션 키를 받도록 수정 (`NOT_FOUND` 에러 해결).
- 위 수정 후에도 "연결됨" 표시와 무관하게 웹소켓이 조용히 죽을 수 있는 구조적 문제 발견
  → 동기화 주기마다 `Client._gateway[sid].is_connected`를 직접 확인해 재연결.
- nexadmin 페이지가 React Hooks 규칙 위반(`useMemo`가 조건부 return 뒤에 있었음)으로
  크래시 → 프로덕션에서 "This page couldn't load"로 나타났던 원인.
- 채팅 미리보기의 `scrollIntoView`가 페이지 전체 스크롤까지 건드려 화면이 튀던 문제
  → 내부 컨테이너의 `scrollTop`만 조정하도록 수정.

### ⚠️ 다음에 이어서 확인해야 할 것
- **출석체크가 실제로 지급까지 되는지 아직 end-to-end로 확인 안 됨.** 채팅 수신 자체는
  확인됐지만(미리보기에 `!출석체크`가 뜸), 지급 로그(`치지직 출석체크 지급: ...`)가 찍히는지,
  실제 포인트/애정도가 오르는지, 치지직 채팅에 "OOO님이 N번째 출석 하셨습니다!" 안내가
  뜨는지는 확인 대기 중.
- 출석체크는 **시청자 인증(`chzzk_verifications`)** 을 마친 계정만 대상 — 스트리머 본인이
  테스트하려면 스트리머 OAuth 연동과 별개로 그 서버의 "입장 인증" 페이지에서 본인 계정으로
  시청자 인증을 한 번 완료해야 함 (자세한 건 이 파일의 CLAUDE.md 아키텍처 섹션 참고).
- `chzzkpy`는 이 프로젝트에서 실사용 검증이 이번이 처음이라, 위에서 잡은 것 외에 또 다른
  라이브러리 내부 이슈가 나올 가능성이 있음. `cogs/chzzk_chat.py`의 로그(`log.info`/
  `log.exception`)를 우선 확인.
- `chzzkpy`가 `aiohttp`/`pydantic` 버전을 정확히 핀 고정해서 설치하므로(`requirements.txt`
  참고), 배포 환경에서 다른 패키지와 버전 충돌이 나면 그쪽을 의심할 것.
