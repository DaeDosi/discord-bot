  바뀌어 있으면 즉시 presence/통계를 재계산.

### 2. Discord 로그인 "잘못된 redirect_uri" 간헐적 오류 (`7840375`)
`/login` 페이지가 `NEXT_PUBLIC_DISCORD_REDIRECT_URI`(프론트 빌드 시점 고정값)로 직접 인증
URL을 만들고, 홈페이지/verify 페이지의 로그인 버튼은 백엔드 `/api/auth/login`
(`DISCORD_REDIRECT_URI`, 서버 런타임 값)을 호출 — 두 값이 어긋나 있으면 **어느 진입점으로
로그인을 시작했는지**에 따라 되기도 하고 안 되기도 했음 (세션 만료로 `/login`에 자동
리다이렉트되는 경우가 특히 잘 걸림). `/login`도 항상 백엔드 값을 쓰도록 통일, 안 쓰는
프론트엔드 환경변수/문서 정리.

### 3. 마크 콜라보 이벤트 — 이번 세션의 핵심 신규 기능 (`b178319` ~ `73a5fd2`)
**요구사항**: 여러 스트리머가 함께하는 마인크래프트 콜라보에서, 각 스트리머 방송 채팅에
`!디버프지급`/`!버프지급`/`!랜덤아이템`을 치면 기존 포인트를 차감하고 공유 마크 서버에
RCON으로 효과를 실행. 디버프는 참가자 중 무작위 1명(자기 제외)에게, 버프는 자기 자신에게.

**아키텍처**: 새 웹사이트를 따로 만들지 않고 기존 nexbot.shop 안에서 해결.
- 설정/카탈로그 관리는 **nexadmin(오너 전용)** — 여러 길드를 가로지르는 크로스길드 개념이라
  개별 스트리머 대시보드에 두기엔 부적합 (RCON 비밀번호 노출, 서버마다 다른 카탈로그로
  밸런스 깨짐 등).
- 실제 구매/발동은 **치지직 실시간 채팅**(`cogs/chzzk_chat.py` 기존 인프라 재사용) — 기존
  `!출석체크`와 동일하게 `chzzk_verifications`로 유저를 매핑하고 `user_points`에서 차감.
- 마크 서버 연동은 **RCON** 직접 구현(`utils/mc_rcon.py`, 외부 의존성 없음) — 봇 프로세스가
  마크 서버로 단방향 명령만 보내면 되므로 플러그인 설치 불필요. RCON 포트는 반드시 봇 서버
  IP만 방화벽 허용해야 함(외부 공개 금지).

**DB**: `mc_events`(이벤트+마크 접속정보, 한 번에 하나만 활성화), `mc_event_guilds`
(참가 서버+마크 플레이어 이름), `mc_event_items`(카탈로그: 효과 명령어+대상 알림 명령어+
채팅 안내 문구, `{player}`/`{user}`/`{item}` 플레이스홀더), `mc_event_commands`(트리거
문구, 이벤트당 자유롭게 추가/이름변경/비활성화 가능 — 이벤트 생성 시 3개 기본값 자동 등록),
`mc_event_purchases`(구매/적용 로그).

**참가 흐름 (B안으로 확정)**: nexadmin에서 오너가 길드를 "초대"(마크 닉네임은 몰라도 됨) →
초대된 길드의 자기 대시보드에 "이벤트 서버 (합방)" 탭이 새로 나타남(초대 안 된 길드에는
안 보임) → 그 서버 관리자가 직접 마크 닉네임 입력, 치지직 연동 상태 확인, 등록된 명령어/
카탈로그 열람(읽기 전용).

**실전에서 잡은 버그들**:
- 참가 서버 삭제가 안 되던 문제 — Discord 스노우플레이크 ID가 JS `number`의 안전 정수
  범위(2^53)를 넘어서 JSON 응답에 숫자로 내려주면 브라우저에서 반올림되어 삭제 요청이 존재
  하지 않는 ID로 나감 → 응답을 문자열로 변경 (이 프로젝트에서 이미 `admin_router.py`가
  같은 이유로 문자열 캐스팅하고 있던 걸 이번 라우터에 놓쳤던 것).
- `chzzk_chat.py` 기존 로직에 잠재 버그: `chzzk_chat_commands`가 하나도 없는 길드는 채팅
  구독 자체를 안 하던 것 → 마크 이벤트만 참가하고 출첵/자동응답은 설정 안 한 길드는 채팅을
  영영 못 받는 문제였음 → mc_event 존재 여부도 함께 확인하도록 수정.
- "미인증 유저"/"아이템 없음"/"트리거 매칭 실패" 케이스가 대부분 조용히 무시되기만 하고
  로그가 안 남아 디버깅이 안 됐음 → 전부 `log.info`로 남기고, 매칭 실패 로그엔 `mc_event`
  자체가 없는 건지(참가 미등록) 트리거만 안 맞는 건지 구분되게 문구 개선.
- 채팅 명령어 이름을 바꾸는 UI가 애초에 없었음(추가/삭제만 가능) → 인라인 편집 추가.
- nexadmin "초대" 판정이 이벤트 `is_active=1`을 요구해서, 초대만 하고 아직 활성화 전이면
  참가자 대시보드에 탭 자체가 안 보이던 문제 → 초대 여부와 활성화 여부를 분리 (활성화는
  실제 채팅 명령어 동작 여부에만 관여).
- `chzzk_chat.py`의 설정 동기화 주기가 5분이라, 스트리머 연동 직후나 명령어/이벤트 설정을

- 기존에 하나의 사이드바 항목("방송설정") 안에서 플랫폼 탭으로 전환하던 구조를,
  사이드바 최상위 항목 3개(치지직/유튜브/숲)로 분리. 유튜브/숲은 준비중 페이지.
- 치지직 페이지 자체에는 "스트리머 설정" / "실시간 채팅 명령어" 세부 탭 신설.

### 6. 치지직 실시간 채팅 명령어 — 이번 세션의 핵심 신규 기능 (`467d01c` ~ `69da220`)
**요구사항**: 치지직 생방송 채팅에서 `!출석체크`(커스터마이즈 가능)를 치면 하루 1회
포인트+애정도를 지급하고, 그 외 최대 5개까지 등록 가능한 커스텀 명령어는 지정한 문구로
자동 응답.

**아키텍처 결정**: 비공식 쿠키 스크래핑 대신 치지직 **공식 오픈 API의 세션(Session)
기능**을 사용 — 이미 스트리머 연동 시 저장해둔 OAuth 토큰을 재사용. 프로토콜(Engine.IO v3
기반, 표준 `python-socketio`로는 안 됨)은 직접 구현하지 않고 이미 이걸 구현해 둔
[`chzzkpy`](https://pypi.org/project/chzzkpy/) 라이브러리를 실제로 설치해 API 시그니처를
검증한 뒤 사용.

**구현물**:
- DB: `chzzk_chat_commands`(guild당 checkin 1개 + reply 최대 5개), `chzzk_checkin_log`
  (1일1회 강제, PK 충돌로 중복 차단), `chzzk_chat_log`(디버그용 최근 50개 채팅 로그),
  `chzzk_subscriptions.chat_last_sync_at`/`chat_last_event_at`(연결 상태 표시용).
- 스트리머 OAuth 스코프에 "채팅 메시지 조회"/"채팅 메시지 쓰기" 추가 — **기존에 연동된
  스트리머는 재연동 필요**.
- `cogs/chzzk_chat.py`(신규): 세션 연결 → 스트리머별 구독 → 채팅 이벤트 수신 → 명령어
  매칭 → 지급/응답. 5분마다 동기화 + 게이트웨이 생존 확인(끊겼으면 강제 재연결).
- 대시보드 "실시간 채팅 명령어" 탭: 명령어 CRUD, 연결 상태 카드, 실시간 채팅 미리보기
  (기본 숨김, 버튼으로 열람 — 실제 수신 메시지/봇 응답을 그대로 보여주는 디버그 도구).
- 입장 인증 탭에 임베드 미리보기 토글 버튼 추가.

**실전에서 잡은 버그들** (라이브 로그 보고 그 자리에서 수정):
- `Client.connect()`의 반환값이 구독 API가 요구하는 세션 키가 아니라 소켓 transport
  ID였음 → `on_connect` 이벤트로 진짜 세션 키를 받도록 수정 (`NOT_FOUND` 에러 해결).
- 위 수정 후에도 "연결됨" 표시와 무관하게 웹소켓이 조용히 죽을 수 있는 구조적 문제 발견
  → 동기화 주기마다 `Client._gateway[sid].is_connected`를 직접 확인해 재연결.
- nexadmin 페이지가 React Hooks 규칙 위반(`useMemo`가 조건부 return 뒤에 있었음)으로
  크래시 → 프로덕션에서 "This page couldn't load"로 나타났던 원인.
- 채팅 미리보기의 `scrollIntoView`가 페이지 전체 스크롤까지 건드려 화면이 튀던 문제
  → 내부 컨테이너의 `scrollTop`만 조정하도록 수정.

### ⚠️ 다음에 이어서 확인해야 할 것
- **출석체크가 실제로 지급까지 되는지 아직 end-to-end로 확인 안 됨.** 채팅 수신 자체는
  확인됐지만(미리보기에 `!출석체크`가 뜸), 지급 로그(`치지직 출석체크 지급: ...`)가 찍히는지,
  실제 포인트/애정도가 오르는지, 치지직 채팅에 "OOO님이 N번째 출석 하셨습니다!" 안내가
  뜨는지는 확인 대기 중.
- 출석체크는 **시청자 인증(`chzzk_verifications`)** 을 마친 계정만 대상 — 스트리머 본인이
  테스트하려면 스트리머 OAuth 연동과 별개로 그 서버의 "입장 인증" 페이지에서 본인 계정으로
  시청자 인증을 한 번 완료해야 함 (자세한 건 이 파일의 CLAUDE.md 아키텍처 섹션 참고).
- `chzzkpy`는 이 프로젝트에서 실사용 검증이 이번이 처음이라, 위에서 잡은 것 외에 또 다른
  라이브러리 내부 이슈가 나올 가능성이 있음. `cogs/chzzk_chat.py`의 로그(`log.info`/
  `log.exception`)를 우선 확인.
- `chzzkpy`가 `aiohttp`/`pydantic` 버전을 정확히 핀 고정해서 설치하므로(`requirements.txt`
  참고), 배포 환경에서 다른 패키지와 버전 충돌이 나면 그쪽을 의심할 것.
